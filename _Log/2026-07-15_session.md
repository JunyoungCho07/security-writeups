## Struggle Points (paste에서 관측)

- **L15 — openssl 노이즈 = "뽀록" 착시**: handshake 진단(cert/session-ticket/`read R BLOCK`)이 화면을 덮어 "왜 됐는지 모름". 실체는 "TLS 입은 nc" + `verify error`는 신원 문제지 암호화 실패 아님. `-quiet`로 노이즈 걷으면 본질 노출.
- **L16 — password 첫 글자 `k`가 KEYUPDATE로 먹힘**: 비-`-quiet` openssl 인터랙티브가 라인 첫 글자를 명령으로 해석(TLS1.3 `k`=KeyUpdate) → password `k…`의 앞글자 누락 → 반복 `Wrong!`. `-quiet`(리터럴 전송)로 해결. 또 privileged port·`nmap -sS`(root 필요) 삽질.
- **L17 — mktemp 파일 vs 디렉토리**: `mktemp` 파일에 `cd`→`Not a directory`(디렉토리는 `-d`). `sort|uniq -u`는 old+new 2줄 반환 → grep 재확인 필요(diff이면 `>` 한 줄).
- **L18 — interactive vs non-interactive**: 로그인 즉시 `Byebye!`(`.bashrc` 트랩) → `ssh host "cmd"`(non-interactive, `.bashrc` 미source)로 우회.
- **L20 — privileged port + 동시성 + 응답 위치**: `nc -l -p 523`→`Permission denied`(포트<1024 root 필요)→9999로. tmux로 2-프로세스. bandit21 password가 **리스너 출력**에 뜬다(suconnect 창 아님)는 걸 처음엔 놓침.
- **RE 세션 — 가설 2회 반증**: group-read 가설(→0400으로 반증), bandit20 파일 가설(→`x/s`가 `.prevpass`로 반증). 증거가 추측을 잡아줌.
