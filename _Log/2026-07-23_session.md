## Struggle Points (정직 기록 — 다음 세션 컨텍스트)

- **L27 포트 문법**: `git clone -p 2220` 반복 실패("unknown switch p"). git엔 포트 플래그 없음 → `:port`는 host 뒤(authority)에. path 끝 `:port`는 **접속 22 + 폴더명=포트**(한 콜론, 두 파서).
- **L31 divergence 소용돌이**: `.gitignore` 편집을 먼저 커밋한 게 씨앗 → fetch/pull/merge/switch/checkout/reset 난동. **핵심 오해**: "push rejected = 동기화 문제"로 착각. 실제론 **pre-receive가 password 출력 후 항상 거부**(golden repo 보존) → 이긴 push도 `remote rejected`. clean clone에서 `add -f`→`commit`→`push` 3줄이면 끝. (unrelated-histories force-update는 **서버측**, 플레이어 탓 아님.)
- **L31 gitignore**: `*.txt`가 `git add key.txt` 막음(무시안내+exit 1) → `git add -f`.
- **L32 근본 이해 재구축(가장 큼)**: `$0`이 왜 sh를 띄우나 → 다단계 Q&A로 **sh -c / system()=execl / argv0 이름표 / setuid·RUID·EUID / bash -p / fork·execve·waitpid / tty·foreground / builtin vs 외부·PATH / ABI·userland·architecture / 소스→바이너리→프로세스**까지 밑바닥부터 재건. C 미경험자 대상 "할머니 버전" + 로컬 데모(minishell.c 컴파일, SHLVL/PID 중첩, isatty, multi-call `$0`)로 못박음. VirtualBox capture ↔ 터미널 foreground 직관 연결.
- **검증 방식**: git 레벨마다 워크플로로 리팩트를 어드버서리 검증(예: verify-pack size 컬럼=delta payload, whole=최대blob, lightweight tag 4-object 지문, quarantine≠rollback, `$0` 고정점).
