## Struggle Points (paste에서 관측)

- **삽질 1 — localhost 차단**: bandit13 서버에서 곧장 `ssh -i ... bandit14`로 다음 레벨 접속 시도 → "Connecting from localhost is blocked" + `~/.ssh` 생성 실패(home read-only). 근본: OWTW가 level→level localhost SSH를 막음 → 키를 **내 머신으로 반출**해 외부 IP로 우회해야 함.
- **삽질 2 — scp 포트 누락**: `scp host:~/sshkey.private .`에 `-P` 미지정 → 기본 22로 접속 → "port 22 not intended" → publickey 거부. 근본: scp의 port는 **대문자 `-P`**(소문자 `-p`는 preserve). flag 에러 없이 조용히 실패 → 발견 지연.
- **삽질 3 — 권한 too open**: scp로 받은 키가 `0640`(group readable) → "UNPROTECTED PRIVATE KEY FILE ... will be ignored" → password로 fallback. 근본: OpenSSH는 `(mode & 077)≠0`이면 키 거부. 사용자는 `chmod 700`으로 해결(통과하나 x 비트 불필요 — **600/400이 정석**).
- **삽질 4 — pass 파일을 dir로 착각**: `cd /etc/bandit_pass/bandit14` → "Not a directory". bandit14는 파일 → `cat`이 정답.
