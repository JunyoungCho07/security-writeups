## [1] Context — 왜 이 작업을 하는가

### Problem
현재 `CLAUDE.md` (vault root) = 11968 chars / ~3000 tokens monolith. 모든 conversation에서 전체 로드됨. 그러나 §10 (Level Structure), §11 (Concept Structure)는 `_Templates/`와 완전 중복이며, §7 (EOL), §13 (Commit), §4 (Frontmatter) 등은 특정 trigger 발동 시에만 필요한데 항상 로드됨 → **lazy load 원칙 위반**.

### Goal
Anthropic Skill pattern (thin `SKILL.md` + lazy-loaded `REFERENCE.md`)을 vault 지침 구조에 적용:
- CLAUDE.md = thin orchestrator (~85 lines, always-loaded only)
- `_System/*.md` = trigger-specific protocol (lazy load via explicit pointer)
- `_Templates/*.md` = file format spec (이미 lazy load 구조, 변경 없음)

### Quantitative Target
- CLAUDE.md: 280 lines → **~85 lines** (70% 감축)
- Persistent token overhead per conversation: 3000 → **~750 tokens** (75% 감축)
- 중복 제거: §10, §11이 `_Templates/`와 중복이었음 → single source of truth 확립

### Scope
- ✅ 건드림: `CLAUDE.md`, `COWORK_PROJECT_INSTRUCTIONS.md`, 신규 `_System/` 폴더
- ❌ 안 건드림: `Wargames/`, `Concepts/`, `Tools/`, `_MOC/`, `_Log/`, `_Templates/`, `scripts/`
- 학습 파일(Level/Concept/Tool 노트)은 monolith 유지 — 사용자가 학습용으로 self-contained 선호
