## Struggle Points (paste에서 관측)

- **L06 — stdin vs argument**: `find … \| cat`/`\| more`/`\| vi`로 경로를 파이프해 "파일이 안 읽힌다" 착각. 파이프는 경로 *문자열*을 stdin으로 넘길 뿐. 정답은 인자(`cat PATH`) 또는 `-exec cat {} \;`.
- **L12 — no-sudo & xxd 방향**: `sudo apt-get install binwalk` → sudo not setuid(root 없음). `xxd`의 `-d`(decimal offset)/`-o`/`-i`를 출력 플래그로 오해 — 정답은 방향 `-r` + 위치인자 `xxd -r IN OUT`. gzip/bzip2 확장자 강제·bzip2 기본 압축 함정.
- **L13 — 내부 hop 불가(내 오답 포함)**: 두 내부 포트 모두 실패 — 2220은 loopback 자원 차단, 22는 시스템 sshd라 bandit14 키 미등록(`publickey denied`). 내가 "포트만 빼라(22로)"고 한 조언이 틀렸고 HINT가 정정. 정답은 키를 로컬 반출 → 외부 IP 2220 접속. (키 크기도 1679B→2602B로 변동 관측 → OTW 키 교체 정황.)
- **L14 — nc stdin/방향/read-only home**: `nc HOST PORT <pw>`(arg→포트 오해), `< <pw>`(파일명 오해), `nc -l`(서버 모드, 방향 반대), 그리고 임시파일 만들려던 `vi/nano`가 home read-only로 실패. 정답은 `nc localhost 30000` 후 stdin 타이핑 / `nc … < /etc/bandit_pass/bandit14`.
